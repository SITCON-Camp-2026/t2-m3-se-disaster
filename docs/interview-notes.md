# 使用者訪談紀錄

> Codex 草稿，待人類刪改、補充與確認。這些回饋來自三個 persona sub-agent，依據 `release-packs/01-interview-kit/` 執行；不是標準答案，也不是最終產品決策。

## Sub-agent 執行紀錄

- 回報者：Wegener sub-agent，依 `reporter.md` 回覆。
- 資訊整理者：Mencius sub-agent，依 `organizer.md` 回覆。
- 行動者：Kant sub-agent，依 `actor.md` 回覆。
- Main agent 只負責整理入檔與標示待人類確認，不把 sub-agent 回饋改成最終決策。

## 我的 Phase 0 prototype 現況

- 目前首頁是一個前端-only 的資訊整理工作台，分成「原始資訊登錄」、「整理工作台作業」、「已整理資訊彙整」三個區塊。
- 使用者可以把 Phase 0 原始資訊送入整理工作台，建立、編輯、刪除、重設與確認整理草稿。
- 畫面有持續標示 `needs_review` / `unverified`，並提醒「整理確認」不代表外部查核完成。
- 工作台目前會記錄候選類型、信心程度、來源角色、下一步、不能直接行動原因與人工確認註記。
- 訪談後的人類修改建議：把「信心程度」改成「原文清楚度」，選項改為「原文模糊 / 原文簡略 / 原文清晰」。
- 目前仍是教學練習資料；不能拿來做真實救災判斷。

## Persona 1：回報者

### 我想完成的事

- 我想很快把我知道或聽到的狀況送出去，不要因為不知道完整地址、數量或分類就不能回報。
- 如果我是轉述別人的訊息，我希望系統能清楚保留「我不是親眼看到」。
- 我想知道我的回報送出後，會不會被當成已確認的事實或任務。

### 我看不懂的地方

- 我分不太清楚「整理確認」和「查核狀態」差在哪裡，看到「已整理資訊」會有點以為事情已經被確認了。
- 「候選類型」「信心程度」「下一步」這些欄位對我來說像內部用語，我不知道是不是要我自己判斷。
- 首頁比較像管理清冊，我不確定自己這種回報者應該從哪裡開始。

### 我最怕的錯誤

- 我只是轉貼群組或家人說的話，結果系統把它當成我親眼確認的現場狀況。
- 我沒有講清楚的地點、數量或需求，被系統整理成很確定的派工資訊。
- 看到「確認列入已整理資訊」後，後面的人直接照做，而沒有再確認現場。

### 我不敢直接相信的資訊

- 社群貼文、群組截圖、別人轉述的地點和物資數量，我都不敢確定是不是最新的。
- 只有「更新時間」但沒有說清楚事情發生時間的內容，我不知道是不是已經過期。
- 來源類別看起來像官方或志工回報時，我也不敢直接相信它一定查證過。

### 我希望系統提醒我的事

- 提醒我可以寫「不知道」「不確定」「只是轉述」，不要逼我猜。
- 提醒我不要把不確定的地點、電話、人物資訊寫得像確定事實。
- 提醒我送出後仍需要人工確認，不代表已經可以直接派人或送物資。

### 我想反問開發者的一個問題

- 如果我只知道一半，或只是看到群組裡有人說，這個系統希望我照樣回報，還是先不要送出？

## Persona 2：資訊整理者

### 我想完成的事

- 我想快速比對原文、整理草稿、資訊取得方式、查核狀態，確認有沒有被整理得太肯定。
- 我想把「能整理」和「能行動」分開，不希望下一位協作者把整理結果當成任務指令。

### 我看不懂的地方

- 「已整理資訊彙整」這個名稱還是有點像已經處理完成，我需要一直提醒自己它不等於 verified。
- 「整理確認」和「外部查核完成」同時出現在畫面時，我會擔心新手分不清楚差別。

### 我最怕的錯誤

- AI 草稿被人工按下確認後，後面的人以為內容已經可以派工。
- 原文只有模糊線索，但摘要寫得太完整，看起來像系統已經補完並確認過。

### 我不敢直接相信的資訊

- 只有資訊取得方式，卻沒有確認時間、確認人或確認依據的資料。
- 來源角色是轉述、第三方或 unknown 的資料，尤其是地點、數量、需求還不完整時。

### 我希望系統提醒我的事

- 在確認草稿前提醒我：「這只是整理確認，不是事實查核完成」。
- 當草稿摘要看起來比原文更確定時，提醒我檢查是否補了原文沒有的資訊。
- 對 `needs_review` / `unverified` 的資料，持續提醒不能直接變成任務。

### 我想反問開發者的一個問題

- 你們希望我按下「整理確認」以前，至少要留下哪些人工判斷理由，才不會讓下一位協作者誤解？

## Persona 3：行動者

### 我想完成的事

- 我想快速知道哪一筆資訊現在可以拿來做下一步，哪一筆只能先等人確認。
- 我想避免跑錯地點、送錯物資，或照著已經過期的資訊行動。

### 我看不懂的地方

- 「已整理資訊」看起來很正式，我會擔心自己把它誤會成「已經查核可以行動」。
- 「信心：高」和 `needs_review` 同時出現時，我不太確定到底該相信哪一個。

### 我最怕的錯誤

- 我看到「確認列入已整理資訊」後，以為這筆已經可以出發處理。
- 地點或物資狀態其實已經變了，但畫面讓我以為仍然有效。

### 我不敢直接相信的資訊

- 只有模糊地標、社群轉述、第三方代講的資訊，我不敢直接照做。
- 雨鞋數量、道路封閉、集合點開放這種會很快變動的資訊，我不敢只看整理摘要就行動。

### 我希望系統提醒我的事

- 請一直提醒我：「整理完成」不等於「事實已確認」。
- 如果資訊有時間、衝突、來源角色不明，請讓我不要忽略。

### 我想反問開發者的一個問題

- 我在這個畫面上，要怎麼一眼分辨「可以行動」和「只能請人再確認」？

## 人類補充

- 待人類確認：以上已改為三個 sub-agent 的正式訪談結果，請至少刪改一處，補上你們實際看畫面後的觀察。
- 三個 sub-agent 共同指出「已整理」或「整理確認」可能被誤解成事實查核完成；是否要在 v1 改成更保守的說法，需要小組決定。
- 回報者特別指出內部欄位語氣太重，像是要求回報者自己判斷「候選類型」「信心程度」「下一步」。
- 行動者特別指出「信心：高」和 `needs_review` 同時出現時會混淆，這應該列入下一階段設計討論。
- 人類修改：後續文件與 v1 設計草稿應以「原文清楚度」取代「信心程度」，避免使用者誤會成資料可信度或可行動程度。
- 人類修改：選項採用「原文模糊 / 原文簡略 / 原文清晰」。這三個詞只描述原始文字是否支撐整理判斷，不代表資料已查核或可以行動。
